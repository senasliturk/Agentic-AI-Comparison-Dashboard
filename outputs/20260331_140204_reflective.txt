- Watch: [REST API Concepts (YouTube – WebConcepts)](https://www.youtube.com/watch?v=Q-BpqyOT3a8) and [Introduction to Requirements Engineering (YouTube – SE University)](https://www.youtube.com/watch?v=2kK59Qz8d6E)

### Day 2: Requirements Engineering Fundamentals  
**Topics:**  
- Requirements elicitation techniques: interviews, surveys, workshops, observation  
- Types of requirements: functional vs. non-functional  
- Writing and documenting clear requirements  
- Common pitfalls and anti-patterns in requirements engineering  
- Collaboration and stakeholder communication basics  

**Activities:**  
- Read chapters 3–4 from *“Software Requirements”* by Karl Wiegers (free summaries available online)  
- Practice: Write clear functional and non-functional requirements for a simple app (e.g., to-do list)  
- Review a sample requirements document and identify issues or ambiguities  
- Watch: [Requirements Elicitation Techniques (YouTube – SE University)](https://www.youtube.com/watch?v=9S9QpZr6p1c)  
- Reflect on communication strategies to handle conflicting stakeholder needs  

---

### Day 3: Core API Design Principles & Hands-on Practice  
**Topics:**  
- RESTful design principles: statelessness, resource modeling, HTTP methods  
- URI design and versioning best practices  
- API documentation standards: Swagger/OpenAPI basics  
- Common API design pitfalls and anti-patterns  

**Activities:**  
- Read: [“REST API Tutorial”](https://restfulapi.net/) (focus on design principles sections)  
- Hands-on: Design a simple REST API (e.g., book catalog) including endpoints and HTTP methods  
- Explore Swagger UI: create a basic OpenAPI spec for your API ([Swagger Editor](https://editor.swagger.io/))  
- Use Postman to send requests to a public API ([Postman Learning Center](https://learning.postman.com/))  
- Practice identifying design issues in sample APIs (provided in resources)  

---

### Day 4: Requirements Validation, Traceability & Change Management  
**Topics:**  
- Validating requirements: reviews, prototyping, automated testing  
- Traceability matrices and their role  
- Managing requirement changes and version control  
- Impact of requirement changes on API design  

**Activities:**  
- Read relevant sections from *“Software Requirements”* by Karl Wiegers (validation and management chapters)  
- Create a traceability matrix linking requirements to API endpoints/features  
- Review case study: analyze how requirement changes affected an API project  
- Practice prioritizing requirements using MoSCoW method (Must-have, Should-have, Could-have, Won’t-have)  
- Watch: [Requirements Validation Techniques (YouTube – SE University)](https://www.youtube.com/watch?v=3oAzuI6cZyU)  

---

### Day 5: Advanced API Topics & Security  
**Topics:**  
- Authentication and authorization (OAuth 2.0, API keys)  
- Error handling and HTTP status codes best practices  
- Rate limiting, throttling, and security considerations  
- Scalability and performance basics  
- API monitoring and documentation quality  

**Activities:**  
- Read: OAuth 2.0 overview ([oauth.net](https://oauth.net/2/)) and API security best practices ([API Security OWASP](https://owasp.org/www-project-api-security/))  
- Practice: Define error response structures and status codes for your API design  
- Use Postman to test error cases and authentication flows with public APIs  
- Review examples of API monitoring tools (e.g., New Relic, Datadog)  
- Watch: [API Security Essentials (YouTube)](https://www.youtube.com/watch?v=QhV6o7vJXGk)  

---

### Day 6: Integrating Requirements with API Design & Testing  
**Topics:**  
- Mapping requirements to API endpoints and functionality  
- Writing API test cases from requirements  
- Iterative design and incorporating feedback  
- Collaboration and communication in iterative development  

**Activities:**  
- Mini-project: Starting from a set of requirements, design an API and write corresponding test cases (unit and integration)  
- Use Postman to create and run test collections based on your test cases  
- Review common testing frameworks for APIs (e.g., Jest, Mocha for Node.js)  
- Get peer or instructor feedback on your design and tests, and refine accordingly  
- Watch: [API Testing with Postman (YouTube)](https://www.youtube.com/watch?v=FjgN6k_GWBY)  

---

- Comprehensive review of all key concepts and glossary terms  
- Practice with sample exam questions and case studies  
- Exam-taking strategies: time management, question analysis, stress reduction  
- Planning for post-exam reflection and continuous learning  

**Activities:**  
- Take at least one full-length practice exam or answer a set of past midterm questions  
- Review mistakes and clarify misunderstandings  
- Group discussion or teach-back session to reinforce learning  
- Prepare a quick-reference sheet with formulas, definitions, and tips for exam day  
- Watch: [Exam Strategy and Stress Management Tips (YouTube)](https://www.youtube.com/watch?v=V1p8Q4y2WZM)
